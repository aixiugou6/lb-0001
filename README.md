# 个人运动计划管理器

一个基于 Node.js + Express + SQLite 的个人运动计划管理工具，支持训练计划的增删改查、筛选、统计，以及实际训练记录的关联管理功能。

## 功能特性

- 📋 **计划管理**：新增、编辑、删除运动计划
- ✅ **完成标记**：一键标记计划完成状态
- 🔍 **多维度筛选**：按类型、日期、完成状态筛选，支持本周快捷筛选
- 📊 **本周统计**：首页展示本周计划数、已完成数、总训练时长
- 🏃 **本月训练统计**：首页展示本月训练次数、总训练时长
- 📝 **训练记录**：为每个计划关联实际训练记录，记录训练日期、实际时长、消耗估算、感受备注
- 📄 **计划详情页**：查看计划完整信息及所有关联训练记录
- 📈 **列表计数**：列表标题实时显示当前筛选结果数量
- 📝 **训练模板**：保存常用训练模板，字段包括训练类型、默认时长、动作说明和备注
- ⚡ **模板快速套用**：创建训练计划时可从模板快速带入信息，一键生成计划
- 💾 **本地存储**：使用 SQLite 数据库（或 JSON 文件），数据持久化
- 🎨 **响应式设计**：适配桌面和移动设备

## 技术栈

- **前端**：原生 HTML / CSS / JavaScript
- **后端**：Node.js + Express
- **数据库**：SQLite（better-sqlite3）

## 快速开始

### 环境要求

- Node.js >= 14.0.0
- npm 或 yarn

### 安装依赖

```bash
npm install
```

### 启动服务

```bash
npm start
```

服务启动后，访问 http://localhost:3000 即可使用。

### 开发模式

```bash
npm run dev
```

### 零依赖版本（备选）

如果无法安装 npm 依赖，可直接运行零依赖版本，功能完全一致，使用 JSON 文件存储数据：

```bash
node server-simple.js
```

服务启动后访问 http://localhost:8080

## API 接口

### 训练计划 API

#### 获取计划列表

```
GET /api/plans
```

查询参数：
- `type`：训练类型筛选
- `date`：日期筛选（精确匹配）
- `completed`：完成状态筛选（true/false）
- `startDate`：开始日期（范围筛选）
- `endDate`：结束日期（范围筛选）

#### 获取本周统计

```
GET /api/plans/stats
```

返回本周计划总数、已完成数、总训练时长。

#### 获取本月训练统计

```
GET /api/plans/monthly-stats
```

返回本月训练记录总数、总训练时长、总消耗卡路里。

#### 获取所有类型

```
GET /api/plans/types
```

#### 获取单个计划详情（含关联训练记录）

```
GET /api/plans/:id
```

返回计划对象，其中 `records` 字段为该计划关联的所有训练记录数组。

#### 新增计划

```
POST /api/plans
```

请求体：
```json
{
  "name": "晨跑5公里",
  "type": "有氧",
  "plan_date": "2024-01-15",
  "duration": 30,
  "completed": false,
  "notes": "状态不错"
}
```

#### 更新计划

```
PUT /api/plans/:id
```

#### 切换完成状态

```
PATCH /api/plans/:id/toggle
```

#### 删除计划（级联删除关联训练记录）

```
DELETE /api/plans/:id
```

### 训练记录 API

#### 获取训练记录列表

```
GET /api/records
```

查询参数：
- `plan_id`：按关联计划 ID 筛选

#### 获取单条训练记录

```
GET /api/records/:id
```

#### 新增训练记录

```
POST /api/records
```

请求体：
```json
{
  "plan_id": 1,
  "training_date": "2024-01-15",
  "actual_duration": 32,
  "calories_burned": 280,
  "feeling_notes": "早晨空气很好，跑完很舒畅"
}
```

字段说明：
- `plan_id`：关联的训练计划 ID（可选，不填则为独立记录）
- `training_date`：训练日期（必填，YYYY-MM-DD）
- `actual_duration`：实际训练时长（必填，分钟）
- `calories_burned`：消耗卡路里估算（可选，千卡）
- `feeling_notes`：感受备注（可选）

#### 更新训练记录

```
PUT /api/records/:id
```

#### 删除训练记录

```
DELETE /api/records/:id
```

### 训练模板 API

#### 获取模板列表

```
GET /api/templates
```

查询参数：
- `type`：训练类型筛选

#### 获取单个模板

```
GET /api/templates/:id

```

#### 更新模板

```
PUT /api/templates/:id
```

#### 删除模板

```
DELETE /api/templates/:id
```

## 数据字段说明

### 训练计划（workout_plans）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键，自增 |
| name | string | 训练名称 |
| type | string | 训练类型（有氧/力量/柔韧/恢复等） |
| plan_date | string | 计划日期（YYYY-MM-DD） |
| duration | integer | 预计时长（分钟） |
| completed | integer | 完成状态（0-未完成，1-已完成） |
| notes | string | 备注信息 |
| created_at | datetime | 创建时间 |

### 训练记录（training_records）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键，自增 |
| plan_id | integer | 关联的训练计划 ID（外键） |
| training_date | string | 训练日期（YYYY-MM-DD） |
| actual_duration | integer | 实际训练时长（分钟） |
| calories_burned | integer | 消耗卡路里估算（千卡） |
| feeling_notes | string | 感受备注 |
| created_at | datetime | 创建时间 |

### 训练模板（training_templates）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键，自增 |
| name | string | 模板名称 |
| type | string | 训练类型（有氧/力量/柔韧/恢复等） |
| default_duration | integer | 默认时长（分钟） |
| exercise_description | string | 动作说明（详细训练动作、组数、次数） |
| notes | string | 备注信息 |
| created_at | datetime | 创建时间 |

## 常见问题

**Q: 端口被占用怎么办？**
A: 修改 `server.js` 中的 `PORT` 变量为其他端口。

**Q: 如何添加更多训练类型？**
A: 在新增/编辑计划时，选择「其他」类型，或直接在数据库中添加。

**Q: 示例数据有多少条？**
A: 首次启动会插入 7 条示例计划数据，覆盖本周每天，包含多种训练类型；同时插入 2 条示例训练记录，分别关联到晨跑和力量训练计划。

**Q: 删除计划会删除训练记录吗？**
A: 是的。删除训练计划时会级联删除所有关联的训练记录，这是为了保证数据完整性。

**Q: 训练记录必须关联计划吗？**
A: 不是必须的。训练记录的 `plan_id` 字段是可选的，可以创建独立的训练记录。但在本应用的 UI 中，训练记录是通过计划详情页进行管理的，因此默认会关联到当前计划。

**Q: 删除模板会影响已创建的计划吗？**
A: 不会。模板只是用于快速生成计划的预设，套用模板后生成的计划是独立的数据，与模板不再有关联关系。删除模板不会影响任何已创建的计划。

**Q: 模板的动作说明会保存到计划中吗？**
A: 是的。套用模板创建计划时，模板的动作说明和备注会自动合并填入计划的「备注」字段中，分别用【动作说明】和【备注】标记区分，方便查看。

**Q: 有多少示例模板数据？**
A: 首次启动会插入 7 条示例训练模板，覆盖常见训练类型：标准晨跑（有氧）、上肢力量训练、下肢力量训练、核心强化（力量）、HIIT燃脂（有氧）、瑜伽放松（柔韧）、游泳训练（有氧）。

### v1.1.0 - 2026-06-21

**Bug 修复：**
- 修复本周统计数量与列表数量不一致的问题
